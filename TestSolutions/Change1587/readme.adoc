== Overview

This document presents a complete test suite for GCBASIC's `TABLE` functionality. The test program exercises virtually all supported table features. Source definitions were cross-verified against the generated assembly.

**Conclusion:** All tables compile correctly with accurate data.

== Key Features Tested

* Decimal, Hex (`0x`, `H'`), Binary (`b'`), and mixed literals
* `AS BYTE` / `AS WORD` declarations
* Automatic type upgrade to WORD
* String handling (single-line, multi-line, CSV merging)
* Escape sequences (`\n`, `\t`, `\r`, `\0`, `\"`, `\\`, `\&nnn`)
* Compile-time arithmetic and `#define` constants
* Leading zeros handling
* Inline comments
* `STORE DATA` (EEPROM) vs Program Memory
* Large tables (stress test)

== Debugging Table Transformations

To troubleshoot how the preprocessor transforms table data, you can enable a specific debug mode in the compiler settings. This will output the **"In"** (source) and **"Out"** (processed ASCII bytes) for each data line to the console.

=== Configuration

Edit the `use.ini` file in your GCBASIC installation directory:

[source,ini]
----
[gcbasic]
; 2048 = cSHOWTABLESTRINGHANDLER
compilerdebug = 2048
----

This debug option is especially useful when debugging complex string tables, escape sequences, or CSV merging behavior.

== Important Behaviors

=== String Merging
Multiple quoted strings on the **same line** are merged without separators.  
**Recommended safe method** for multiple strings:
```gcbasic
Table MyStrings AS BYTE
    "String1", 0
    "String2", 0
    "String3", 0
End Table
```

=== Leading Zeros
Treated as **decimal**, not octal (`008` = 8).

=== Storage Options
* Default → Program Memory (Flash)
* `STORE DATA` → EEPROM
